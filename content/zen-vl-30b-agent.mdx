zen-vl-30b-agent is the vision-language entry in the Zen line: a mixture-of-experts model that takes images and text together, for agents that read a screen or a document instead of a string. It is superseded by [zenlm/zen3-vl](https://huggingface.co/zenlm/zen3-vl), the canonical name for this slot. The repo stays published so existing references and reproductions keep working.

## Lineage

Repackaged from [Qwen/Qwen3-VL-30B-A3B-Instruct](https://huggingface.co/Qwen/Qwen3-VL-30B-A3B-Instruct) (Apache-2.0), built by Qwen at Alibaba Cloud. This is a redistribution for the OSS-clean Zen line. We did not train it from scratch and we did not fine-tune it. The vision-language capability is Qwen's work.

From this repo's `config.json`: `Qwen3VLMoeForConditionalGeneration`, model type `qwen3_vl_moe`, with a vision config and a text config. Unlike some of the older Zen repos, the name and the weights agree here: this one really is vision-language, and really is MoE.

## Use it

```python
from transformers import AutoModelForImageTextToText, AutoProcessor

model_id = "zenlm/zen-vl-30b-agent"
processor = AutoProcessor.from_pretrained(model_id)
model = AutoModelForImageTextToText.from_pretrained(
    model_id, torch_dtype="auto", device_map="auto"
)

messages = [{
    "role": "user",
    "content": [
        {"type": "image", "url": "https://huggingface.co/datasets/huggingface/documentation-images/resolve/main/pipeline-cat-chonk.jpeg"},
        {"type": "text", "text": "What is in this image?"},
    ],
}]
inputs = processor.apply_chat_template(
    messages, add_generation_prompt=True, tokenize=True,
    return_dict=True, return_tensors="pt"
).to(model.device)
print(processor.batch_decode(model.generate(**inputs, max_new_tokens=256)))
```
