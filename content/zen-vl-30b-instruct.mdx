zen-vl-30b-instruct is the largest vision-language checkpoint in the Zen line. Images and video frames go in, text comes out: captioning, document and screenshot OCR, chart reading, visual question answering. It is a sparse mixture of experts, so only a subset of the weights is active per token. Its `config.json` declares 128 experts across 48 text layers, a 262,144-token position limit, and a 27-layer vision encoder. Long documents in one pass is the reason to reach for this over the dense 8B.

## Lineage

This repository redistributes [Qwen/Qwen3-VL-30B-A3B-Instruct](https://huggingface.co/Qwen/Qwen3-VL-30B-A3B-Instruct) (Apache-2.0), trained and released by Alibaba's Qwen team. The weights are their work and the credit is theirs.

The safetensors are byte-identical to upstream. All thirteen shards carry the same SHA-256 as the Qwen release: no additional training, no merge, no abliteration. We publish it under the Zen name so our catalog and serving stack have one stable identifier with explicit provenance, and so the Zen line stays permissively licensed end to end. Apache-2.0 upstream, Apache-2.0 here. If you are already pulling the Qwen repo, you have this model.

## Use it

```python
from transformers import AutoProcessor, AutoModelForImageTextToText

model_id = "zenlm/zen-vl-30b-instruct"
processor = AutoProcessor.from_pretrained(model_id)
model = AutoModelForImageTextToText.from_pretrained(
    model_id, dtype="auto", device_map="auto"
)

messages = [{
    "role": "user",
    "content": [
        {"type": "image", "url": "https://example.com/invoice.png"},
        {"type": "text", "text": "Extract every line item and its total."},
    ],
}]

inputs = processor.apply_chat_template(
    messages, add_generation_prompt=True, tokenize=True,
    return_dict=True, return_tensors="pt",
).to(model.device)

out = model.generate(**inputs, max_new_tokens=512)
print(processor.decode(out[0][inputs["input_ids"].shape[1]:], skip_special_tokens=True))
```
