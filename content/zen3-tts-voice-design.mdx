zen3-tts-voice-design starts from a sentence about a voice rather than a recording of one. Describe the speaker you want, in words, and it synthesizes someone who fits: an older man with a gravelly voice and a slight drawl, a bright young narrator, a tired night-shift dispatcher. Nobody has to be recorded, which makes it the honest option for game characters, audio drama, and prototypes where cloning a real person would be the wrong move.

The description is the interface, so it carries the detail: timbre, age, accent, pace, and mood all belong in the instruction. The config marks this variant `voice_design`, and like [zen3-tts-custom-voice](/blog/zen3-tts-custom-voice/) it carries no speaker encoder, since there is no reference clip. Weights are about 1.92B parameters in bf16 over the same 12 Hz codec as the rest of the line. One practical note: a designed voice is generated fresh each call. When you need one character to stay consistent across many lines, synthesize a short clip you like, then feed that clip to [zen3-tts](/blog/zen3-tts/) as a reference and generate the rest from there.

## Use it

```bash
pip install -U qwen-tts
```

```python
import torch
import soundfile as sf
from qwen_tts import Qwen3TTSModel

model = Qwen3TTSModel.from_pretrained(
    "zenlm/zen3-tts-voice-design",
    device_map="cuda:0",
    dtype=torch.bfloat16,
)

wavs, sr = model.generate_voice_design(
    text="Storm's coming in off the water. Best tie that down.",
    instruct="An older fisherman, gravelly low voice, unhurried, slight coastal accent.",
    language="English",
)
sf.write("out.wav", wavs[0], sr)
```
