`zen-eco-4b-instruct` is a small dense instruct model for the large fraction of production work that does not need a frontier model: summarizing a document, extracting fields into JSON, classifying a ticket, rewriting text, or routing a request to something bigger. Plenty of pipelines send jobs like these to a frontier model and pay frontier prices on every request out of habit.

Eco is the size that fits on one modest GPU, which changes where you are willing to put a model. Per-request latency and cost stop being the reason you cache aggressively or batch overnight. It ships with GGUF and MLX conversions in the repository for llama.cpp and Apple silicon.

## Lineage

Fine-tuned from [Qwen/Qwen3-4B-Instruct-2507](https://huggingface.co/Qwen/Qwen3-4B-Instruct-2507) (Apache-2.0) by Alibaba's Qwen team. The architecture and pretraining are Qwen's work, and the credit belongs to them. The weights here still carry Qwen3-4B's shapes exactly.

Hanzo adds identity training, agentic-data fine-tuning, and abliteration. That is the whole list. Nothing was trained from scratch and there is no Zen architecture under it.

## Use it

```python
from transformers import AutoModelForCausalLM, AutoTokenizer

model_id = "zenlm/zen-eco-4b-instruct"
tok = AutoTokenizer.from_pretrained(model_id)
model = AutoModelForCausalLM.from_pretrained(model_id, dtype="auto", device_map="auto")

messages = [{"role": "user", "content": "Extract every version number in this changelog as JSON."}]
ids = tok.apply_chat_template(
    messages, add_generation_prompt=True, return_tensors="pt"
).to(model.device)

out = model.generate(ids, max_new_tokens=256)
print(tok.decode(out[0][ids.shape[-1]:], skip_special_tokens=True))
```
